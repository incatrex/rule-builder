Demonstration of how JsonEditor now automatically adds placeholder UUIDs for new rules to enable validation

üéØ Problem & Solution

Problem

Previously, new rules without UUIDs couldn't be validated in the JsonEditor because the validation schema requires a UUID field.

Solution

JsonEditor now automatically detects new rules (without UUID) and adds a placeholder UUID for validation purposes. The placeholder is clearly marked and will be replaced by the server when the rule is saved.

üö´ Result: Schema validation fails due to missing UUID

‚úÖ Result: Schema validation passes with placeholder UUID

üîÑ User Experience Flow

Step 1: New Rule Created

User creates a new rule in RuleBuilder. The rule has no UUID yet (will be server-generated).

Step 2: Switch to JSON Editor

User switches to JSON Editor tab to view/edit the JSON. JsonEditor detects missing UUID and automatically adds a placeholder.

üîµ PLACEHOLDER UUID Notice Appears:
"A temporary UUID has been added for validation. The server will generate the final UUID when the rule is saved."

Step 3: Validation Works

User can now validate the rule successfully in JsonEditor because the placeholder UUID satisfies the schema requirements.

Step 4: Save Rule

When rule is saved, the placeholder UUID and flag are automatically removed. Server generates the final UUID.

Detection Logic

// Detects if rule needs placeholder UUID if ((data.structure || data.metadata) && (!data.uuId || data.uuId === null)) { // Add placeholder UUID for validation return { ...data, uuId: generatePlaceholderUUID(), __placeholderUUID: true // Flag for tracking }; }

Cleanup Logic

// Removes placeholder flags before saving const removePlaceholderFlags = (data) => { const cleaned = { ...data }; if (cleaned.__placeholderUUID) { delete cleaned.__placeholderUUID; // UUID itself is removed by RuleService } return cleaned; };

üîß Implementation Complete

The JsonEditor component has been enhanced with the following new functions:

  • generatePlaceholderUUID() - Creates temporary UUIDs for validation
  • addPlaceholderUUID() - Detects and adds placeholder UUIDs when needed

The enhancement is backward compatible and doesn't affect existing rules with valid UUIDs. The feature seamlessly integrates with the new service-based architecture and server-side UUID generation.
